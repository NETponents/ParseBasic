ParseBasic Documentation

ParseBasic is a programming language designed to make it easier to create parsers for the Octoduino project. The syntax is created so the code is very easy to understand by anyone who has done coding before.

Boot process

ParseBasic emulates an SD card root at the location where it is run. It first searches for a bootloader folder. Inside, it will look for a file called bootloader.pba. All ParseBasic files have the .pba extension. Execution will begin inside of that file.

Command set

ParseBasic has a very basic command set. They usually follow this syntax: COMMAND arg1 arg2 ... ; Make sue that the command is ALWAYS in caps. Otherwise you will recieve a tokenizer error.

PRINT (v0.1.0- )

Syntax: PRINT "string to write";

Notes:

NEWPRINT (v0.1.0- )

Syntax: NEWPRINT;

Notes: Creates a new line in output.

WAIT (v0.1.0- )

Syntax: WAIT int;

Notes: int is in millisecods.

CREATESWAP (v0.1.0- )

Syntax: CREATESWAP;

Notes: This is emulated in the ParseBasic x86-64 interpreter. This creates the SWAP folder (if it doesn't already exist).

IO (v0.1.0- )

Syntax: IO int1 int2;

Notes: int1 is the port, int2 is the status (high or low).

NEW (v0.1.0- )

Syntax: NEW string1 string2;

Notes: string1 is the name of the variable to create. string2 is the initial value.

DELETE (v0.1.0- )

Syntax: DELETE string;

Notes: string is the name of variable to delete.

SET (v0.1.0- )

Syntax: SET string1 string2;

Notes: string1 is name of variable. string2 is value to set.

EXTLOAD (v0.1.0- )

Syntax: EXTLOAD string;

Notes: string is file path to .pba file relative to the (emulated) SD root.

FILEWRITE (v0.1.0- )

Syntax: FILEWRITE string1 string2;

Notes: string1 is file to write to relative to the (emulated) SD root. string2 is the string to write.

FILEREAD (v0.1.0- )

Syntax: FILEREAD string1 string2;

Notes: string1 is file to read from relative to the (emulated) SD root. string2 is the name of the variable to write the read buffer to.

FILERM (v0.1.0- )

Syntax: FILERM string1;

Notes: string1 is the name of the file to delete relative to the (emulated) SD root.

END (v0.1.0- )

Syntax: END;

Notes: Ends execution of the program.
